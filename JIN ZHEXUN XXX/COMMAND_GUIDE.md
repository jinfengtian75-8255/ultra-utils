# 🚀 사장님을 위한 핵심 명령어 가이드 (UltraUtils)

이 가이드는 사장님이 혼자서도 사이트를 배포하고 관리하실 수 있도록 꼭 필요한 명령어들만 정리해둔 **비밀 수첩**입니다.

---

### 1. 내 컴퓨터에서 미리보기 (개발 모드)
수정 사항을 라이브에 올리기 전에, 사장님 컴퓨터에서만 미리 확인할 때 사용합니다.
*   **명령어**: `npm run dev`
*   **사용법**: 터미널에서 입력 후 엔터 → 브라우저에서 `localhost:3000` 접속
*   **용도**: 기능이 잘 작동하는지, 디자인은 예쁜지 개인적으로 테스트하는 용도입니다.

### 2. 라이브 사이트에 반영하기 (배포 모드) 🌟 **(가장 중요!!)**
사장님 컴퓨터에서 수정한 내용을 전 세계 사람들이 보는 진짜 사이트에 업로드할 때 사용합니다.
*   **명령어**: 
    1. `cd utility-tools` (폴더가 다를 경우 먼저 이동)
    2. `npx vercel --prod --yes`
*   **설명**: 
    *   `--prod`: 진짜 '라이브(Production)' 서버에 올리라는 뜻입니다.
    *   `--yes`: 일일이 묻지 말고 즉시 실행하라는 뜻입니다.
*   **팁**: 글자 하나라도 고친 뒤에는 꼭 이 명령어를 쳐야 세상에 반영됩니다.

### 3. Vercel 로그인 다시 하기 (보안 인증)
배포 시 "Token is invalid" 같은 에러가 나면서 주인을 못 알아볼 때 사용합니다.
*   **명령어**: `npx vercel login`
*   **사용법**: 
    1. 명령어 입력 후 엔터
    2. 화살표 키로 `Continue with Google` 선택 후 엔터
    3. 다시 한번 엔터 키를 눌러 브라우저 열기
    4. 브라우저에서 **[Allow]** 버튼 클릭
*   **용도**: 약 2~4주마다 한 번씩 보안상 로그인을 다시 해주어야 배포 권한이 유지됩니다.

### 4. 만약 엉뚱한 주소로 배포가 된다면? (프로젝트 연결)
분명 배포를 했는데 `ultra-utils.com`에 반영이 안 되고 다른 주소만 바뀐다면, 연결을 다시 확인해야 합니다.
*   **명령어**: `npx vercel link`
*   **사용법**: 
    1. `npx vercel link` 입력
    2. `Link to existing project?` 라고 물으면 `Y` (Yes) 입력
    3. `Which scope?` 라고 물으면 사장님 계정 선택
    4. `Link to "ultra-utils"?` 라고 물으면 `Y` (Yes) 입력
*   **용도**: 내 컴퓨터 폴더가 어떤 인터넷 주소(방)랑 연결될지 다시 정해주는 작업입니다. 현재는 **ultra-utils.com**으로 꽉 묶어두었으니 평소에는 안 하셔도 됩니다!

### 💡 꿀팁: 터미널이 멈춘 것 같다면?
*   **Ctrl + C**: 현재 실행 중인 명령어를 강제로 취소합니다. (명령어를 잘못 쳤거나 터미널을 청소하고 싶을 때 사용하세요.)
*   **새 터미널 열기**: 터미널 창 우측 상단의 `+` 버튼을 누르면 깨끗한 새 도화지가 생깁니다. 꼬였을 때는 새 창에서 하는 게 가장 빠릅니다.
